## Структура проекта

- **`docker-compose.yml`** – оркестрация сервисов (MySQL, бэкенд, фронтенд).
- **`backend/`** – FastAPI + SQLAlchemy + PyMySQL.
- **`frontend/`** – React 18 + TypeScript + Vite.
- **`init.sql`** – создание таблиц и наполнение тестовыми данными.
- **`reset_db.sh`** – скрипт для полного сброса и пересоздания БД.

## Запуск

### 1. Убедитесь, что Docker работает

```bash
docker --version
docker compose version
```

### 2. Клонируйте или перейдите в корень проекта

```bash
cd test_task
```

### 3. Запустите все сервисы одной командой

```bash
docker compose up -d
```

> При первом запуске произойдёт скачивание образов `mysql:8.0`, `python:3.12-slim`, `node:18-slim` (иногда может падать с ошибкой, в таком случае попробуйте использовать vpn)

### 4. Проверьте работу

- **Swagger (бэкенд)**: http://localhost:8000/docs
- **Фронтенд**: http://localhost:5173
- **Прямое подключение к MySQL** (при необходимости):  
  ```bash
  docker exec -it test_mysql mysql -u <пользователь> -p<пароль> <имя_бд>
  ```

## Основные страницы фронтенда

- **Все задачи** – `/tasks` (канбан-доска со всеми задачами)
- **Мои задачи** – `/my-tasks` (задачи, где текущий пользователь исполнитель; можно менять статус)
- **Мои созданные** – `/my-created-tasks` (задачи, где текущий пользователь автор; можно менять исполнителя, приоритет, удалять, созадвать новые задачи)
- **Аналитика** – `/analytics` (графики среднего времени и топ-продуктивных пользователей)
